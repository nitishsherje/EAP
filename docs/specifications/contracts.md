# §4 — EAP Specification Contract

All resources share envelope `apiVersion: eap.crisil/v1` + `kind` + `metadata` + `spec`.  
Implementation base: `src/eap/specifications/envelope.py` (`EapResource`, `Metadata`).  
Loader: `loader.py` (`load_yaml`, `load_file`, `parse_resource`).

---

## AgentSpec — IMPLEMENTED

| | |
| --- | --- |
| Purpose | Autonomous reasoning entity (distinct from Workflow) |
| Location | `specifications/agent.py` — `AgentSpec`, `Agent` |
| Example | `contracts/examples/auditor-report-agent.yaml` |
| Important fields | `model`, `instructions`/`prompt`, `skills`, `capabilities`, `knowledge`, `guardrails`, `output_schema`, `memory`, `max_iterations` |
| Validation | Scheme validators on refs; semantic checks in `SpecificationService._validate_agent` |
| References | `model://`, `skill://`, `capability://`, `knowledge://`, `policy://`, `prompt://`, `schema://` |
| Lifecycle | draft → publish via LifecycleService; resolve as root |

## WorkflowSpec — IMPLEMENTED (pattern enum broader than runtime)

| | |
| --- | --- |
| Purpose | Coordinate agents/skills/steps |
| Location | `specifications/workflow.py` — `WorkflowSpec`, `Workflow`, `WorkflowStep` |
| Example | `contracts/examples/rating-note-workflow.yaml` |
| Important fields | `pattern`, `steps[]` (`id`, `type`, `ref`, `inputs`, `depends_on`), `output_targets`, `on_error` |
| Validation | Unique step ids; dep existence; cycle detection in SpecService |
| References | Step refs must match type scheme |
| Runtime note | `pattern` values include parallel/iterative/dynamic but **WorkflowStrategy ignores pattern** and uses topo-order only |

## SkillSpec — IMPLEMENTED

| | |
| --- | --- |
| Purpose | Reusable procedure that uses capabilities (not itself a tool) |
| Location | `specifications/skill.py` — `SkillSpec`, `Skill`, `CapabilityUse` |
| Example | `contracts/examples/auditor-extraction-skill.yaml` |
| Important fields | `type` (deterministic/agentic/…), `capabilities[]` (`ref`, `operation`), `knowledge`, `inputs`/`outputs` |
| Validation | Deterministic without capabilities → warn; agentic without model → warn |
